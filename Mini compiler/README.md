# Mini C Subset Compiler (Flex + Bison)

This project is a mini compiler for a small subset of C.

It performs:
1. Lexical analysis (Flex)
2. Parsing + AST construction (Bison)
3. Semantic checks (undeclared use, duplicate declaration)
4. Three Address Code (TAC/IR) generation
5. C code generation from IR (`output.c`)

## Project Files

- `lexer.l`: Defines token rules for keywords, identifiers, numbers, operators, braces, and punctuation.
- `parser.y`: Defines grammar rules, builds AST nodes, and performs semantic checks during parsing.
- `ast.h`: Declares AST node types, IR structures, enums, and shared function prototypes.
- `symbol_table.c`: Implements AST constructor/free functions and symbol table operations.
- `ir_gen.c`: Traverses AST and generates Three Address Code (TAC/IR), then prints IR.
- `codegen.c`: Converts generated IR instructions into equivalent C code (`output.c`).
- `main.c`: Driver program that runs parse -> semantic check -> IR generation -> code generation.
- `sample_input.txt`: Example input program you can compile immediately.

## What Each File Does (Detailed)

### Hand-written source files

- `lexer.l`
  - Input to Flex.
  - Converts source text into parser tokens (`INT`, `IF`, `IDENTIFIER`, `NUMBER`, etc.).
  - Tracks line numbers for error reporting.

- `parser.y`
  - Input to Bison.
  - Recognizes the C subset grammar.
  - Creates AST nodes for declarations, assignments, expressions, blocks, `if`, and `while`.
  - Performs semantic checks:
    - duplicate variable declaration
    - use of undeclared variables

- `ast.h`
  - Core shared header.
  - Defines:
    - AST node kinds and operator enums
    - IR instruction structure
    - function declarations used across modules

- `symbol_table.c`
  - Keeps a simple global symbol table.
  - Provides insert/lookup/reset/print symbol table helpers.
  - Also contains AST creation/free logic used by parser actions.

- `ir_gen.c`
  - Walks AST and emits TAC-like IR instructions.
  - Handles temporary variable and label generation.
  - Prints IR in readable form.

- `codegen.c`
  - Reads IR list and emits equivalent C statements with labels and gotos.
  - Writes generated code to `output.c`.

- `main.c`
  - Opens input program file.
  - Invokes parser.
  - Stops on parse/semantic errors.
  - Calls IR generator, prints IR, and writes generated C output.

### Auto-generated or runtime files

- `lex.yy.c`
  - Generated by `flex lexer.l`.
  - C implementation of the lexer.

- `parser.tab.c` and `parser.tab.h`
  - Generated by `bison -d parser.y`.
  - Parser implementation and token/header definitions.

- `mini_compiler.exe`
  - Final executable generated by GCC.

- `sample_input.txt`
  - Example input program for testing compiler features.

- `output.c`
  - C code generated by your mini compiler from IR.

## Supported C Subset

- Integer declarations: `int x;`
- Assignments: `x = 5;`
- Arithmetic expressions: `+ - * /`
- Relational operators: `< > <= >= ==`
- `if`, `if-else`
- `while`
- Blocks with `{ ... }`

## How It Works

1. `lexer.l` tokenizes source code into tokens.
2. `parser.y` parses tokens and builds AST nodes.
3. During parsing, semantic checks are performed:
   - duplicate declaration
   - undeclared variable usage
4. If no semantic errors exist, `ir_gen.c` generates TAC IR.
5. IR is printed to console.
6. `codegen.c` writes equivalent C code to `output.c`.

## Build Instructions

The generated files `lex.yy.c` and `parser.tab.c` are already included, so you can build the compiler directly with `gcc`.

```bash
gcc lex.yy.c parser.tab.c main.c symbol_table.c ir_gen.c codegen.c -o mini_compiler.exe
```

## Run

```bash
.\mini_compiler.exe sample_input.txt
```

After running:
- IR/TAC is printed in terminal.
- Generated C code is written to `output.c`.

## Small Test Program

Use `sample_input.txt` with:

```c
int x;
int y;
x = 10;
y = x + 5;

if (y >= 15) {
    x = y * 2;
} else {
    x = y - 1;
}

while (x > 0) {
    x = x - 1;
}
```

Then run:

```bash
.\mini_compiler.exe sample_input.txt
```

## Notes

- The current symbol table is global (single scope), so redeclaring a variable anywhere is reported as duplicate.
- Only integer variables are supported in this implementation.
